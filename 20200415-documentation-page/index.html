Java Introduction

Java identifiers are words like class, public, if, else, boolean, etc.; they can, however, be defined by the user. Examples include names for classes, methods, and variables. The names can have letters, digits, underscores, and $--they cannot begin with digits, however. The $ is usually reserved for mechanically generated source code.

Java identifiers with pre-set meanings cannot be again used as identifiers—they are reserved keywords. Examples:

Java Literals

What the program is merely passing through—e.g. to print out. A fixed value; compare to an expression using variables. Starting from Java 7, underscores can be used to make the literal more readable.

An integer literal is automatically assumed to be of type int; to mark it as long, one has to include at the end of the number an ‘L’ (or lowercase ‘l’, although uppercase is encouraged for readability). An integer literal is by default decimal; to make it hexadecimal, preface it with a zero and an x—0X or 0x. To make it binary, preface with 0b, or 0B. Floating point numbers are double by default; float can be used by appending f or F, and double can be marked with d or D.

Scientific notation can be used with e or E to stand for the exponent—e.g. 1.475 x 10^2 is 1.475E2 [or E+2]; 1.475 x 10^, -2 is 1.475E -2. Underscores can be used between numbers in numerical literals.

Control Flow

Basic Conditionals

Keyword 'if' introduces a conditional statement. Curly brackets or braces are used to set aside a block, for more than one statement after a conditional. Format:

if (Boolean expression) {
Statements;
}

Additional Control: Providing an “else if" or "else" alternative to “if.”

if (Boolean expression) {
Statement;
} else if (Boolean expression) {
Statement;
} else {
Statement;

Additional if/else constructions can be nested within “else” or “if.”

if (Boolean expression) {
if (Boolean expression) {
Statement;
} else (Boolean expression) {
Statement;
} } else if (Boolean expression) {
Statement;
} else {
Statement;
}

Example of conditional statements to determine if a year is a leap year:

if(year%4!=0) return false;
else if(year%100!=0) return true;
else if(year%400!=0) return false;
else return true;

Ternary Operator

The ternary operator: the “eval” function. Short for "if-then-else". To evaluate using the ternary operator, need a function with three arguments, in the nature of f(b, e1, e2). B stands for boolean, and e1 and e2 can be of any type so long as they are the same between them. The process can be expressed as: b? e1: e2. And so there are, in the case of the ternary operator, two options preceded by a condition: if the first argument is true, the second argument (and only the second) is activated (the third is not touched); if the first argument is false, then the third argument (and only the third) is activated.

Loops

While Loop

A loop that is first primed with a true condition if it is to be used, and then repeats a body of code within it until the boolean proves false due to a condition within it. A break keyword will break out of the loop; a continue keyword will skip the next iteration. The while statement/loop can be looped in other such statements by nesting. Bart Burd says that "most five-layered loops are hidden within method calls" (Burd, 2014).

while (diskScanner.hasNext ()) { statements; }

For Loop

The For loop sets a variable that is modified every recurrence and compared to another variable. When the comparison returns false, the loop ends. For loop structure is as follows:

for (Initialization; Condition/termination; Update/increment) { Statements; }

Do...While Statement

do { Statements } while (condition);

Enhanced For Loop (For-each)

Format:

for (<TypeName> <variableName> : <RangeOfValues>) { Statement(s); }

Example:

int [] toFive = {1,2,3,4,5}; for(int i : toFive) { System.out.println("Your number is " + i); }
Classes and Objects

Classes and Object-Oriented Programming

Java is object-oriented. OOP always begins with a class. Java programs need a class—every Java program must have at least one class. Class names begin with an upper-case letter. In Java, we take two ways of description—behavior (methods) and states (fields, known in Java and other languages as variables)—and encapsulate them with the fitting language constructs in objects. Objects of the same kind belong to the same class. An object's states can be accessed from without the class, if the proper permission is granted.

A class is set off with curly braces {} and is usually indented to make it easier for humans to examine and work on. A class is created like so:

class Classname { [list of attributes:] attributetype attributename; ... [list of methods] public/private static/etc. void/returntype methodName(){attribute = x;} ... }

A class can be created in it's own .java file (recommended), in the Main.java file but outside of the main class, or even created and initialized at once inside the main class as an "inline" (this is used in Android apps, commonly for "click listeners").

Earlier, we talked about primitive types like char, int, boolean, and double (see BBurd, ch. 6,7,8). Earlier languages were limited to these simple types and had to break down complex processes with multiple different data types and values into simple parts. Newer languages like Java can create a type of basket in which simpler types/values are combined into what can be thought of as a new, composite "type/value." This is called a "reference type" (BBurd, 2014, p. 335). When all the parts of a process are kept in one integral construction, tracking and repair are easier.

Objects

Objects "come from" classes. Classes are the plans, objects are the ends. Objects have attributes and methods that were specified in their class. Classes describe how the object will be created in a main method which is called the constructor—how pieces of data are initialized, combined, and used. A constructor has three characteristics:

  1. Must be named as the class is
  2. Must not have a return type—including void
  3. Must be invoked with the new operator in the creation of an object; it then initializes an object.

Constructors may be overloaded like other methods are. This allows one to initialize objects in different ways. Usually, a constructor without arguments is provided. This is called a no-arg or no-argument constructor. If no constructor is made in the class definition, the Java compiler will make one automatically, with no arguments. This is implicit definition—a public no-arg constructor with no body is provided.
